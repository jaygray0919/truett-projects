SDL Proposed Design

How we achieved SDL proposed design (1)

  • We needed to have different formats for input. We concluded that we can convert graph into different formats like, rdf/xml, rdfa, turtle, json-ld.
  • Now we have all these formats, there is just a need to modify linter views.

How we achieved SDL proposed design (2)

  • As the implementation of views is in key/value paris like,
    {"snippet": "...", "html": "...", "messages": [], "statistics": {"count": 9, "templates": []}}
    We passed "rdf":"rdfformat", "rdfa": "rdfaformat", "ttl": "ttlformat", "jsonld": jsonldformat", "graph": graphformat".
  • For graph, we currenly use http://www.ldf.fi/service/rdf-grapher API.

SDL as a service on AWS Lambda (2)

SDL deployment with AWS Lambda has few limitations:

  • AWS Lambda supports only deployment unzipped size of 250 MB, whereas our SDL software size is >333MB. We remove few unwanted examples, directories and make it to fit in lambda deployment.
  • API Gateway Endpoint timeout is 29 seconds. After 29 seconds, it time out that means it listens to Lambda response only upto 29 seconds. It can process small amount of data currently.
