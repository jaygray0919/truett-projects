2. Ontomatica application content is coded in RDF. Application content is exposed as RDFa on web pages. Web page content may be harvested in Turtle and/or JSON-LD format. For example, application content on this site is published in RDFa & harvested in both Turtle & JSON-LD format.

3. Ontomatica knowledge representations (a.k.a. application content) are coded in RDF.





4. Many instances of Ontomatica application content are published as graphs (Directed Acyclic Graphs). Graphs serve as a form of Proof about logical integrity of application design. For example, several graphs are published on this site to demonstrate scope & completeness of an application model. Graphs are generated from knowledge representations & published in Scalable Vector Graphics (SVG) format.

Image: Author/Organization





6. Ontomatica uses applications & services that implement Rule Interchange Format (RIF). For example, using RIF, a services accepts an OWL file as input & produces a new output file. Examples on this site are OWL-coded ontologies, including Ontomatica ontologies, generated in human-readable format.

Image: Author/Organization





8. Ontomatica uses the PROV ontology to document provenance of information used in our applications. Provenance often is prominently featured in production applications (see header in faceted search applications as examples). Also, we use PROV to establish provenance for methods & complementary chemical, food & drug terms — often encoded in OWL files.
