What is an Ontology?

  1. A model of (some aspect of) the world
  2. Introduces vocabulary relevant to the domain of the model
    • Often includes names for classes and relationships
  3. Specifies intended meaning of vocabulary
    • Typically formalized using a suitable logic
  4. Consists of two parts
    • Set of axioms describing the structure of the model
    • Set of facts describing some particular concrete situation

Axioms describe the Structure of the Model

Class: HogwartsStudent
    EquivalentTo: Student and attendsSchool
                  value Hogwarts
Class: HogwartsStudent
    SubClassOf: hasPet only (Owl or Cat or Toad)
ObjectProperty: hasPet
    Inverses: isPetOf
Class: Phoenix
    SubClassOf: isPetOf only Wizard

Facts describe particular Concrete Situations

Individual: Hedwig
    Types: Owl
Individual: HarryPotter
    Types: HogwartsStudent
    Facts: hasPet Hedwig
Individual: Fawkes
    Types: Phoenix
    Facts: isPetOf Dumbledore

Database Analogy

  1. Ontology axioms analogous to database schema
    • Schema describes structure-of and constraints-on data
  2. Ontology facts analogous to database data
    • Instantiates schema
    • Consistent with schema constraints

Database Query Answering

  1. Schema plays no role
    • Data must explicitly satisfy schema constraints
  2. Query answering amounts to model checking
    • "look-up" against the data

Ontology Query Answering

  1. Ontology axioms play a powerful and crucial role
    • Answer may include implicitly derived facts
    • Can answer conceptual as well as extensional queries
    • Example:
    2.       Can a Muggle have a Phoenix for a pet?
  2. Query answering amounts to theorem proving
